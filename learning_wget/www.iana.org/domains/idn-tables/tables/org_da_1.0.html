This document provides a description of the IDN (Internationalized Domain Names) Language Table and Policy to be used by the .ORG TLD registry for the registration of Danish language domains. The policy is derived based on the Dansk Internet Forum (DIFO) policies for the .DK registry (Denmark), and is consistent and compliant with the ICANN IDN Registry Implementation Guidelines. This document is intended to be published publicly and registered at the IANA Internationalized Domain Names Character Tables Registry.

The following Character Inclusion Table will be used for the .ORG registry. Note that only the "sanitized" character based on the Nameprep [RFC3491] specifications are included. It is expected that implementers first "sanitize" an IDN according to the Nameprep specifications before applying this table.

The table will use the convention U+XXXX to denote Unicode codepoints for each of the accepted characters for IDN registrations. The table includes entry rows with 2 columns: 1. Unicode Codepoint; and 2. Description; each, separated by a semicolon: ";".
